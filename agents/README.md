# LangChain Core Concepts

## Temperature

In LangChain, `temperature` is a model setting that controls how random or creative the model's output is.

Think of it as a creativity dial:

| Temperature | Behavior |
|---|---|
| `0` | Very deterministic, focused, and repeatable |
| `0.2` | Mostly consistent, with slight flexibility |
| `0.7` | Balanced creativity and reliability |
| `1.0+` | More varied and creative, but sometimes less predictable |

Use a lower temperature when you want:

- factual answers
- code generation
- data extraction
- consistent formatting
- reproducible behavior

Use a higher temperature when you want:

- brainstorming
- creative writing
- naming ideas
- storytelling
- alternative phrasings

## StrOutputParser()
The `StrOutputParser()` is a fundamental component in the LangChain library used to convert the raw output of a Large Language Model (LLM) or Chat Model into a simple, clean string.
**Key Features**
* Converts AIMessage to String: If a chat model returns an AIMessage object, this parser extracts the content string.
* Pass-through for Strings: If the model already returns a string, it passes it through unchanged.
* Streaming Support: It supports streaming, allowing it to yield text chunks as they are generated by the model.
* Simplest Parser: It is the most basic parser, designed for when you do not need structured data (like JSON) but just the raw text response.

**Other features for Output Parsers**
- Parse JSON, list, Pydantic models.
- Handle parsing errors gracefully.
- Enable downstream processing.
